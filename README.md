# Руководство по сборке и использованию чекера лабораторных работ (Модуль Docker)

Данное руководство описывает шаги по компиляции приложения `Z69` на новом ПК, настройке окружения (включая решение проблем с Docker и WSL на старых версиях Windows) и тестированию 10-й лабораторной работы.

---

## Часть 1. Компиляция приложения в PyCharm

Чтобы превратить исходный код в один кликабельный файл `.exe` на новом компьютере, внимательно выполните следующие шаги:


2. **Откройте проект**
   - Запустите установленный **PyCharm** (ярлык появится на рабочем столе).
   - В стартовом окне нажмите кнопку **Open** (или в верхнем левом углу верхнего меню: `File` -> `Open...`).
   - В появившемся окне найдите и выберите папку `Z69` (ту самую, где лежат все файлы проекта), и нажмите **OK**.

3. **Установите зависимости**
   Чтобы программа могла работать, ей нужны дополнительные библиотеки.
   - Посмотрите в левую часть экрана PyCharm (там находится список файлов). Найдите файл с названием `install_deps.bat`.
   - Кликните по нему **правой кнопкой мыши** и в появившемся меню выберите пункт **Run 'install_deps.bat'** (он с зеленым треугольником).
   - Внизу экрана откроется черное окно. Подождите, пока там пробегут строчки загрузки и появится надпись `Установка завершена`. После этого нажмите любую клавишу, чтобы окно закрылось.

4. **Скомпилируйте приложение**
   - Посмотрите в **самый низ экрана** PyCharm. Там есть горизонтальная панель. Найдите на ней кнопку **Terminal** (Терминал) и кликните по ней левой кнопкой мыши.
   - Внизу откроется командная строка. Кликните туда мышкой, чтобы появился мигающий курсор, напечатайте команду:
     ```bash
     python build.py
     ```
     и нажмите клавишу **Enter**.
   - Пойдет процесс сборки, он займет 1-2 минуты. В конце появится надпись `completed successfully`.
   - Теперь посмотрите снова в левую панель файлов. Там появилась новая желтая папка `dist`. Откройте её — внутри лежит готовый файл **`Z69.exe`**. Это и есть ваша программа!

## Часть 2. Установка Docker и настройка WSL (Решение проблем со старой Windows)

Чтобы приложение `Z69.exe` могло проверять работы студентов по Docker, на самом компьютере проверяющего должен стоять сам Docker. Если у вас Windows 10 старой версии, обычная установка выдаст ошибку. Делайте строго по этим шагам:

### Пошаговая установка с обходом проблем старой WSL:

1. **Включение компонентов Windows вручную**
   - Нажмите **правой кнопкой мыши** по меню **"Пуск"** (иконка Windows в левом нижнем углу экрана).
   - В появившемся меню найдите и нажмите на **Windows PowerShell (администратор)**. Обязательно "администратор"! Подтвердите запуск, если система спросит разрешения.
   
   - В синем окне кликните мышкой, скопируйте эту команду, вставьте её (кликнув правой кнопкой мыши по синему фону) и нажмите **Enter**:
   ```powershell
   dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
   ```
   - Подождите, пока полоска дойдет до 100%. Затем скопируйте, вставьте и запустите (через Enter) вторую команду:
   ```powershell
   dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
   ```
   - Дождитесь 100%. Закройте синее окно.

2. **Обязательная перезагрузка**
   Обязательно **перезагрузите компьютер** прямо сейчас, иначе следующие шаги не сработают.

3. **Обновление ядра Linux (Ручной метод)**
   - После перезагрузки скачайте файл обновления системы от Microsoft: Скачать пакет обновления WSL 2 (нажми, чтобы скачать).
   - Найдите скачанный файл `wsl_update_x64.msi` в загрузках и запустите его. 
   - В появившемся окне нажимайте "Next", пока установка не закончится (это займет 5 секунд), и нажмите "Finish".

4. **Переключение на WSL 2**
   - Снова нажмите **правой кнопкой мыши** по меню **"Пуск"** и выберите **Windows PowerShell (администратор)**.
   - Вставьте команду и нажмите **Enter**:
   ```powershell
   wsl --set-default-version 2
   ```
   - Должна появиться надпись "Для получения сведений...". Закройте синее окно.

5. **Установка Docker Desktop**
   - Скачайте сам Docker: Скачать Docker Desktop (нажми, чтобы скачать установщик). Установочный файл весит около 600 МБ.
   - Запустите скачанный файл.
   - В самом первом окне установки убедитесь, что стоит галочка **Use WSL 2 instead of Hyper-V** (Она должна стоять!). Нажмите "OK".
   - Дождитесь конца установки и нажмите "Close and restart" (если попросит) или просто закройте окно.
   - Найдите на рабочем столе ярлык **Docker Desktop** и запустите его. Согласитесь с лицензией (Accept). 
   - В левом нижнем углу программы должна появиться зеленая полоска или зеленый значок кита (надпись `Engine running`). **Пока Docker Desktop открыт и горит зеленым, приложение Z69 может проверять работы.**

## Часть 3. Проверка 10-й лабораторной работы

Файлы для 10-й лабораторной работы уже заранее подготовлены и находятся прямо внутри вашей папки с проектом `Z69`. Они лежат в папке, которая так и называется — `10` (внутри неё лежат файлы `Dockerfile`, `docker-compose.yml` и т.д.).

1. **Убедитесь, что файлы на месте**
   Просто проверьте, что внутри главной папки вашего проекта `Z69` лежит папка `10`. Больше ничего скачивать и распаковывать для этого шага не нужно!

2. **Запустите программу проверки (чекер)**
   Зайдите в папку проекта `Z69`, откройте папку `dist` и запустите файл `Z69.exe`.

3. **Откройте консоль Docker в приложении**
   Посмотрите в самый левый край окна приложения `Z69`. Там есть столбец с кнопками-иконками. Нажмите на **вторую иконку сверху** (с изображением синего кита). Выедет окно консоли.

4. **Укажите номер лабораторной работы**
   Кликните левой кнопкой мыши в самом низу выехавшей консоли (там находится черное поле ввода с мигающим курсором). Напечатайте на клавиатуре команду:
   ```text
   lab lab10
   ```
   и нажмите клавишу **Enter**. Вы увидите ответ: `OK: lab=lab10`.

5. **Укажите путь к файлам студента**
   - Снова посмотрите на левый столбец с иконками. Нажмите на **третью иконку сверху** (в виде желтой папки).
   - Откроется стандартное окно Windows для выбора папки. Найдите папку с вашим проектом `Z69`, откройте её, выделите внутри неё папку `10` одним кликом мыши и нажмите кнопку **"Выбор папки"** внизу.
   - В консоли должно появиться сообщение `Docker project path successfully set to...` с указанием пути к вашей папке `10`.

6. **Запустите проверку**
   Снова кликните в поле ввода в самом низу консоли (где мигает курсор), напечатайте короткую команду:
   ```text
   check
   ```
   и нажмите клавишу **Enter**.
   
7. **Анализ результатов**
   - Программа начнет собирать виртуальную среду. В консоли появится мигающий синий прогресс-бар с надписью `[Технические процессы]`. Это займет 1-2 минуты (или быстрее).
   - Затем программа автоматически запустит тесты.
   - В конце вы увидите отчет:
     - **Зеленым цветом** `[PASSED]` — если студент сделал этот пункт задания правильно.
     - **Красным цветом** `[FAILED]` — если студент допустил ошибку.
   - В самом низу будет строка `Success Rate` — процент успешно выполненной работы (например, 100%).
